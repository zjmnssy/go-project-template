# internal目录说明
- 私有应用程序和库代码目录。此牡蛎下的代码你不希望其他人在他们的应用程序或库导入。注意，这个布局模式是由Go编译器本身强制执行的。更多的细节可参考go 1.4发行说明(https://golang.org/doc/go1.4#internalpackages)。请注意，您并不局限于顶级内部目录。在项目树的任何级别上都可以有多个内部目录。  
- 您可以选择向内部包添加一些额外的结构，以分离共享和非共享的内部代码。这不是必需的(特别是对于较小的项目)，但是有可视化的线索来显示打算使用的包是很好的。实际的应用程序代码可以放在/internal/app目录(例如/internal/app/myapp)，而这些应用程序共享的代码可以放在/internal/pkg目录(例如/internal/pkg/myprivlib)。
